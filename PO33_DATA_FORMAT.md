# PO-33 Data File Format (.bin)

This document describes the structure of the binary data files generated by `po33_converter.py`. These files are a lossless representation of the data encoded in PO-33 KO backup audio files.

## High-Level Structure

The `.bin` file contains interleaved data from the **Left** and **Right** audio channels of the stereo backup. Each byte represents 4 phases of the PSK signal.

| Offset | Content | Description |
| :--- | :--- | :--- |
| `0x00` | `L_Byte 0` | First byte from the Left channel |
| `0x01` | `R_Byte 0` | First byte from the Right channel |
| `0x02` | `L_Byte 1` | Second byte from the Left channel |
| `0x03` | `R_Byte 1` | Second byte from the Right channel |
| ... | ... | ... |

The file ends when the shortest channel's data is exhausted.

## Physical Layer & Encoding

The data is extracted from a Differential Phase Shift Keying (DPSK) signal with the following characteristics:

- **Carrier Frequency:** 7800 Hz
- **Symbol Rate:** 7800 symbols per second
- **Bits per Symbol:** 2 bits (4 phases)
- **Total Bitrate:** 15,600 bits per second (per channel)
- **Aggregate Bitrate:** 31,200 bits per second (Stereo)

### Phase Mapping

Each 7800Hz cycle corresponds to one of four phases. For decoding, we look at the first two "bits" of the 4-bit per-cycle bitstream (where bits are defined by zero-crossing timing).

| Phase Label | Bit Pair (Internal) | Value (2-bit) | Description |
| :--- | :--- | :--- | :--- |
| **C** | `00` | `0` | 180° Shift |
| **D** | `01` | `1` | 270° Shift |
| **B** | `10` | `2` | 90° Shift |
| **A** | `11` | `3` | 0° Shift (Baseline) |

## Byte Packing (LSB-First)

Data bits are grouped into bytes using **Least Significant Bit (LSB) first** ordering. One byte is composed of 4 consecutive phases (8 bits).

**Byte Construction:**
`Byte = (Phase3 << 6) | (Phase2 << 4) | (Phase1 << 2) | (Phase0)`

- **Phase 0:** First phase received in time.
- **Phase 3:** Fourth phase received in time.

## Data Interpretation

The resulting binary data contains the compressed sample data and pattern metadata used by the PO-33. 
- **Samples:** Typically 8-bit µ-law companded, 23437.5 Hz samplerate.
- **Header:** The raw audio backups include a sync preamble which is stripped during the `.bin` conversion; however, the `po33_reconstitute.py` script re-generates this preamble (2 seconds of 'A' phases) to satisfy the PO-33 hardware.

## Known Limitations

- **Length Discrepancy:** The converter skips leading silence/noise and stops at the end of the signal. The `.bin` size depends on the actual duration of the backup signal detected in the WAV.
- **Interleaving:** If one channel is significantly longer than the other (due to recording noise), the shorter channel determines the end of the interleaved `.bin` file.
